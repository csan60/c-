# College Student Mental Health Assessment Management System

这是一套基于 **[uni-app](https://uniapp.dcloud.io/)** 框架实现的跨端应用，目标平台覆盖 H5、微信小程序以及原生 App。项目围绕大学生心理健康主题提供测评、咨询、内容浏览及社交互动等功能，支持管理员与普通用户两种角色。

---

## 🎯 核心功能概览

| 模块 | 说明 |
| --- | --- |
| 用户体系 | 登录、注册、忘记密码、修改密码、个人中心 |
| 心理测评 | 在线试卷、题库管理、考试记录、结果查询 |
| 心理健康内容 | 心理健康文章、收藏、留言、论坛互动 |
| 咨询服务 | 咨询师列表、在线预约、聊天记录 |
| 通知公告 | 官方通知发布及展示 |
| AI 咨询 | 基于配置的简单 AI 咨询入口 |

所有页面注册在 `pages.json` 中，并通过 tabBar 和普通路由进行跳转。

---

## 📁 目录结构速览

| 路径 | 作用 |
| --- | --- |
| `App.vue` | 全局入口文件，注册公共样式与生命周期 |
| `main.js` | 挂载全局工具、API 及权限方法，启动应用 |
| `pages/` | 各业务页面（登录、首页、心理测评、论坛等） |
| `components/` | 公共组件（如 `uni-ui`、二维码、评分组件等） |
| `api/` | 接口封装：基础配置、HTTP 请求封装、业务 API |
| `utils/` | 通用工具：菜单权限、校验、导航、主题样式等 |
| `assets/` | 静态资源（样式、图片等） |
| `static/` | 静态图片及 tabBar 图标 |
| `colorui/` | [ColorUI](https://github.com/weilanwl/ColorUI) 主题库 |
| `uni_modules/` | 通过 `uni-app` 插件市场安装的三方组件 |
| `manifest.json` | 多端打包配置、APPID、权限等 |
| `pages.json` | 页面路由、导航栏、tabBar 配置 |
| `uni.scss` | 官方提供的主题变量，可用于统一样式 |
| `unpackage/` | 构建输出目录（应忽略版本控制） |

---

## 🔌 API 层

- `api/base.js`：统一维护后端基础 URL
- `api/http.js`：二次封装 `uni.request`，支持 Token、响应拦截及通用错误提示
- `api/index.js`：提供登录、注册、增删查改、分页、上传等业务接口方法

> 建议把后端地址改为环境变量，避免生产环境误用本地地址。

---

## 🛠️ 工具函数

- `utils/utils.js`：页面跳转、提示弹窗、时间生成等
- `utils/validate.js`：邮箱、手机号、身份证等校验正则
- `utils/system.js` & `utils/menu.js`：本地维护的角色 / 菜单权限，用于控制前端按钮权限

## ✅ 快速开始

> ⚠️ 仓库缺少 `package.json`，建议补充依赖声明。以下命令示例以 HBuilderX 项目为参考：

```bash
# 安装依赖（如存在 package.json）
npm install

# 运行到 H5
dcloud dev:h5

# 运行到微信小程序
# 使用 HBuilderX 打开项目，选择“运行到小程序模拟器”
```

### 构建 / 发布

通过 HBuilderX 选择目标端进行打包，构建产物位于 `unpackage/` 目录。

## 🚀 优化建议

1. **依赖管理 & 构建脚本**
   - 新增 `package.json`，明确依赖与脚本，方便 CI/CD。
   - 引入 `npm`/`yarn` 包管理，统一三方库版本。

2. **环境配置**
   - 将 `api/base.js` 的后端地址改为 `.env` 管理，区分开发、测试、生产环境。
   - 支持多端读取不同环境变量。

3. **代码质量**
   - 引入 ESLint + Prettier，规范代码风格。
   - 考虑 TypeScript 提升可维护性和类型安全。

4. **接口封装**
   - 丰富 `http.js` 的错误处理与重试逻辑。
   - 在响应拦截器里统一处理未登录（401）、权限不足等场景。

5. **安全性**
   - Token 建议加密或采用更安全的存储方式。
   - 关键操作增加二次确认或验证码机制。

6. **页面性能**
   - 大量页面/功能集中在单个项目中，可考虑模块化分层。
   - `pages.json` 较长，可以拆分配置或引入自动生成脚本。

7. **文档**
   - 补充后端接口文档、部署指南、角色权限说明等。
   - 对 `uni_modules` 中安装的插件进行整理和说明。

8. **UI/UX 优化**
   - 统一视觉风格：配色、字体、按纽统一规范。
   - 提升无数据状态、加载状态、错误反馈等体验。

9. **AI 咨询**
   - 当前只是占位页面，可结合实际 AI 服务（如科大讯飞、OpenAI 等）实现智能问答。

10. **自动化测试**
    - 引入单元测试（如 vitest）与端到端测试，提高改动的可靠性。
